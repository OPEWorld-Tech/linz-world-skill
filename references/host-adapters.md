# 宿主适配

## 通用原则

- 优先使用远程 MCP；本地脚本只做配置、身份上下文、调用包装和样板链路辅助。
- 不默认暴露宿主文件系统、浏览器会话、私有仓库、环境变量或密钥。
- 写动作必须使用幂等键；高风险动作必须先 dry-run。
- 最终用户回复只展示业务摘要，不展示原始协议返回。

## Codex

- 推荐注册可复现、无副作用的能力。
- 本地代码修改、shell 执行、文件读取不应注册为 Linz 对外能力，除非用户明确要求。
- 高风险工具必须展示确认摘要后再执行。
- 建议使用 `confirmationRef`，避免把确认令牌原文放入对话或日志。

## Claude Code

- 默认只暴露代码分析、文档生成、测试建议、架构审查等非敏感能力。
- 如需对外注册本地执行能力，必须先说明风险并等待用户确认。
- 不读取或保存私有密钥、浏览器会话或本地个人文件。

## Cursor

- 推荐把本 Skill 作为项目内文档与脚本包导入。
- MCP 地址与 Origin 配置应放在宿主安全配置区，设备授权结果只保存在本地 `memory/`。
- 多窗口并行执行写动作时，应使用不同幂等键和请求文件。

## OpenClaw

- 推荐在安装后注册远程 MCP server。
- 如宿主支持定时任务，可只注册只读状态检查和审计查询。
- 巡航类自动动作不得绕过 Linz 高风险确认边界。

## 通用服务端 Agent

- 每个用户会话应缓存独立 `MCP-Session-Id`。
- 服务端保存设备授权凭证时必须使用密钥管理或加密存储。
- 调用结果应记录请求 ID、业务对象、状态和脱敏摘要。
